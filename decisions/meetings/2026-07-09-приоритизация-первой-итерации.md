## Первая итерация — 16 задач с критериями приёмки

### S1 — Модели данных (11 ORM-сущностей)
**FR:** FR-0/1/2/3/4/5/8/10
**AC:** Миграция `alembic upgrade head` создаёт таблицы, соответствующие data-model.md §1 (11 сущностей: system_user, repository, lesson, artifact_def, edge_def, rubric, git_credential, sync_run, sync_run_repository, artifact_snapshot, coherence_verdict, override). Все колонки и типы совпадают со спецификацией.

### S2 — store.py
**FR:** FR-0/1/3/8/10
**AC:** Для каждой журнальной сущности экспортирована `register_*`, но не `update_*` или `delete_*`. Экспортированы ровно 4 `update_*`: `update_sync_run_status`, `update_override_revoked`, `update_user_lockout`, `update_credential_validity`.

### S3 — Миграции + инварианты DDL
**FR:** все
**AC:** Все CHECK-constraints из ARCHITECTURE §4 (И1, И3, И4, И6, И8, И9, И11 + UNIQUE-части И10) проходят на сиде. `alembic downgrade` откатывает без ошибок.

### I1 — main.py, base.html, login.html, CSS
**FR:** —
**AC:** `uvicorn app.main:app` стартует без ошибок. GET /login возвращает HTML-форму. GET / без аутентификации редиректит на /login.

### S4 — config.yaml + config_manager
**FR:** FR-2
**AC:** При старте `config_manager` читает YAML, создаёт/обновляет записи Lesson, ArtifactDef, EdgeDef, Rubric в БД append-only. Изменение YAML + `POST /admin/reload-config` создаёт новые версии рубрик, не удаляя старые.

### S5 — Аутентификация
**FR:** FR-0
**AC:** POST /login с верным паролем создаёт сессию. POST /login с неверным паролем 5 раз подряд блокирует на 15 минут. GET /logout завершает сессию.

### S6 — CSV-импорт
**FR:** FR-1
**AC:** POST /import-csv с CSV «ФИО,repo_url»: создаёт записи Repository; сводка N доступно / M недоступно / K дубликатов. Повторная загрузка с новым студентом добавляет его, не теряя историю старых.

### G1 — git_client
**FR:** FR-3, NFR-3/4
**AC:** Читает дерево репозитория и содержимое файлов через GitLab API и GitHub API по read-only токену. При rate-limit паузит и повторяет. При протухшем токене возвращает `auth_failed`. Ошибка одного репозитория не валит остальные.

### G2 — sync_orchestrator
**FR:** FR-8, FR-4
**AC:** POST /sync за < 2 мин обходит 25 репозиториев. Каждый артефакт классифицирован: found / partial / not_found. Исходы репозитория: ok_changed / ok_unchanged / repo_unavailable / auth_failed / skipped_rate_limit. Создаётся SyncRun со статусом completed|partial|failed.

### G3 — Детект заготовок (template_copy)
**FR:** FR-4 (BR-3)
**AC:** Артефакт, чей content_hash совпадает с content_hash любого файла из репозитория-шаблона, получает `partial_reason = ["template_copy"]`. Хеши шаблона тянутся раз за обход.

### G4 — Свод-реконсиляция
**FR:** FR-5, FR-8
**AC:** В конце каждого обхода все пары (EdgeDef × репозиторий с текущими снапшотами обеих ролей) без валидного вердикта идентифицированы. На каждую — `asyncio.create_task`. Deferred-пары и потерянные при рестарте задачи автоматически перепроверяются следующим сводом.

### D1 — matrix_builder + matrix.html
**FR:** FR-4
**AC:** GET / показывает матрицу «репозиторий × занятие». Каждая ячейка: статус (found/partial/not_found). Partial_reason отображается. «Актуально на ЧЧ:ММ» присутствует.

### I2 — cron + health
**FR:** FR-8
**AC:** cron 2×/сут дёргает POST /sync. GET /health возвращает время последнего обхода, количество пар без вердикта, количество deferred по причинам.

### D4 (min) — карточка студента, 2 ребра
**FR:** FR-9
**AC:** По студенту — хронология артефактов с разрывами по 2 рёбрам (PRD→схема данных, схема данных→архитектура). Каждый разрыв: вердикт, уверенность, ≤5 точек с цитатами. Хронология по observed_at, не по git-истории.

### O1 — Override (store)
**FR:** FR-10
**AC:** `update_override_revoked(id)` проставляет `revoked_at`. Повторный вызов снимает отметку. Удаления нет — только мягкое гашение.

### O2 — UI отметок
**FR:** FR-10
**AC:** На точке разрыва — кнопка «ложный разрыв». Нажатие создаёт Override. Повторное нажатие снимает. Отмеченный разрыв не подсвечивается при следующих обходах, пока пара документов не изменилась.
