# ADR-004 — LLM-модель ядра: deepseek-v4-flash

- **Статус:** Proposed (станет Accepted после мини-эвала)
- **Дата:** 2026-07-09
- **Решили:** CEO (по итогам митинга упрощения архитектуры, `../../digital-twin/staging/митинг-упрощение-архитектуры-2026-07-09.md`)

## Решение

Модель ядра FR-5 — **`deepseek-v4-flash`**, режим non-thinking (правило-рубрика не требует reasoning-режима).

## Отвергнутые варианты

- **`deepseek-v4-pro`** — плюс: качество reasoning на сложных рёбрах; минус: ~$19/мес по стандартному тарифу — риск бюджета D36 ($20 на всё). Остаётся fallback, если пилот Фазы 0 покажет пропуски на сложных рёбрах.
- **`deepseek-chat` / `deepseek-reasoner` (старые имена)** — плюс: работают сегодня; минус: имена устаревают 2026-07-24, а смена имени модели = смена `llm_model` в четвёрке Б1 = инвалидация всех вердиктов и полный прогон golden set. Захардкодить устаревающее имя — заложить регрессию с известной датой.

## Цена

≈ **$1.3–1.5/мес**: ~900 пар/мес × (~10K токенов вход + ~1K выход); V4-Flash — $0.14/M вход (cache-miss), $0.28/M выход, cache-hit $0.0028/M (закреплённая рубрика кешируется). Тариф: <https://api-docs.deepseek.com/quick_start/pricing/>. Комфортно в бюджете D36.

## Критерий приёмки (гейт Proposed → Accepted)

Совпадение с эталонными вердиктами golden set (`evals/golden-set.md`) + ≤20% ложных разрывов на 2 реальных репозиториях (С-01, С-02).

## План мини-эвала

Прогнать GS-1, GS-2 + 4–6 пар из реальных репо (С-01, С-02) на `deepseek-v4-flash`; зафиксировать вердикты и `llm_model` в golden set. Совпадает с прогоном рубрики Фазы 0.

## Последствия

- `llm_model` входит в четвёрку идентичности вердикта (Б1): смена модели = инвалидация вердиктов = **полный прогон golden set** (железное правило CLAUDE.md).
- Golden set обязан фиксировать `llm_model` каждого прогона (поле добавлено 2026-07-09).
