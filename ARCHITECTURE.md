# ARCHITECTURE — Course Dashboard

> **Статус: не спроектирована.** Стек и архитектура — открытый вопрос №2 PRD (`product/prd.md` §12);
> закрывается отдельной архитектурной сессией с CEO. Результат сессии — этот файл + ADR о выборе стека в `decisions/`.

## Что уже зафиксировано требованиями (не отменяется выбором стека)

- **Компоненты (эскиз из `product/overview.md` §8):** клиент Git API (GitLab + GitHub) · парсер `.md` · анализатор связности — LLM-агент по версионированным рубрикам (FR-5, ADR-001; выбор модели — на этой сессии) · хранилище состояния · веб-дашборд за аутентификацией.
- **Периметр ПД (рамка CEO):** максимум данных у преподавателя; именной реестр не покидает его контур и не уходит в LLM (R-PD3).
- **Модель данных — time-series, не снапшот:** история статусов по занятиям со ссылкой на `source_commit_sha` (PRD §13). Логическая схема — `data-model.md` (занятие 6).
- **Надёжность:** инкрементальный синк, rate-limit/backoff, частичный успех не валит обход (NFR-1..4).
- **Периметр:** токен read-only, шифруется, не в логах; где крутится инструмент — открытый вопрос №5 (152-ФЗ, R-PD3).
