# Персона: Марат, специалист по безопасности ЦУ

**Сегмент:** Служба безопасности / compliance
**Архетип:** «Хранитель периметра»

## Контекст
Марат, 38 лет. Руководитель отдела ИБ и комплаенс в ЦУ. В его ведении — все цифровые сервисы, обрабатывающие данные студентов: LMS, корпоративный портал, интеграции с внешними API. Каждый новый инструмент проходит через его согласование.

В теме 152-ФЗ разбирается глубоко — это его основная зона ответственности. Видел десятки проектов, которые «забыли» про персональные данные на этапе дизайна, а потом переписывали.

К курсу агентной разработки относится нейтрально, но к любому инструменту, собирающему данные о студентах, — с повышенным вниманием.

## Интересы
- Убедиться, что дашборд не обрабатывает и не хранит персональные данные (ФИО, email, git-аккаунт)
- Проверить, какие данные передаются через GitLab/GitHub API и где они хранятся
- Получить документальное подтверждение: описание обработки данных, схему потоков

## Опасения
- Дашборд начнёт собирать данные о студентах «чуть-чуть», а потом обрастёт новыми функциями — и поедет по рельсам нарушения
- Персональные данные утекут через публичный дашборд (репутационный удар по ЦУ)
- Интеграция с GitHub/GitLab передаёт данные на сервера за пределами РФ
- Инструмент запустят без его согласования — потом разбираться ему

## Отношение к технологиям
Высокий уровень понимания. Работает с SIEM, DLP, системами управления доступом. Ко всем новым разработкам относится с презумпцией нарушения безопасности.

## Голос
Чёткий, без лишних слов, с лёгким оттенком усталости от «новаторов, которые не думают о безопасности». Любимая фраза: «а вы это с юристами согласовали?». Часто использует 152-ФЗ как главный аргумент.

## Триггеры интереса к инструменту
- Любой новый сервис, который обращается к внешним API (GitLab/GitHub) — автоматчиеская проверка
- Публичный дашборд (любая информация, доступная без аутентификации) — красный флаг
- Упоминание «прогресс студентов» — сразу запрос: какие именно данные собираются
