# JTBD — Course Dashboard

> Канонический список jobs. Источник: синтетические интервью (`interviews/`), сведение рассинхрона нумерации — `user-stories/01-user-stories-преподаватель.md` §4. Слайд-версия для занятия — `course/занятие-04-демо/слайд-jtbd.md` (там историческая нумерация J1–J5 — здесь каноническая).
> Формула: *Когда* [триггер] → *хочу* [job] → *чтобы* [результат]. Job ≠ фича.

## Канонические JTBD преподавателя (v1 обслуживает только их)

- **J1 (ядро):** *Когда* готовлюсь к занятию, *хочу* видеть, где у студентов порвалась связность артефактов (требования ↔ данные ↔ архитектура), *чтобы* вмешаться точечно, а не обходить 25 репо вручную. — функц. + эмоц.
- **J2 (защита):** *Когда* студент на защите, *хочу* доказательную базу его пути по артефактам, *чтобы* оценка была обоснованной, а не «препод придирается». — функц. + соц.
- **J3 (механика):** *Когда* нужно понять состав сданного по группе, *хочу* автоматическую сверку с чек-листом, *чтобы* не обходить 25 репо вручную и не вводить ничего руками.

## Jobs других ролей (вне v1 — паркуем осознанно, см. PRD §9)

- **J-D · Дамир (портфолио):** *Когда* показываю себя работодателю, *хочу* собрать прогресс курса в цельный кейс, *чтобы* доказать компетенцию. — соц. + эмоц. *(в слайде — J3)*
- **J-A · Алия (приватность):** *Когда* отстаю, *хочу* приватно видеть «что мне осталось», *чтобы* догнать без публичного сравнения с группой. — эмоц. + соц. *(в слайде — J4)*
- **J-S · Студент, любой (приёмка):** *Когда* сдал артефакт, *хочу* понимать, принят он или нет, *чтобы* не угадывать по чужим репо. — сильнейший кросс-персона сигнал корпуса (Г7); требует студенческого интерфейса → v2. *(в слайде — J5)*

## Центральный конфликт (методический нерв проекта)

Один и тот же экран — разные jobs: для преподавателя матрица «студент × задача» решает job контроля; для Алии та же матрица **ломает** её job (догнать без публичности). Разрешение в v1 — радикальное: студенческого интерфейса не существует (BR-4).
