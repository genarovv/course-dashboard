---
name: meeting
description: Провести совещание ролей проекта над одним решением (приоритизация,
  скоуп, архитектурная развилка). Загружает роли из roles/, выбирает состав ролей,
  чьи темы пересекаются с вопросом, структурированное несогласие, развилки — человеку по одной.
  Use when решение имеет конфликт критериев; NOT для фактических вопросов
  (цены/лимиты — это задача проверки фактов) и не для задач с одним правильным ответом.
---

Прочитай `roles/совещание.md` и проведи совещание строго по этому протоколу.

Уточнения для Claude Code:
- Развилки фазы 3 задавай через AskUserQuestion — по одной, с вариантами и ценой в description.
- Артефакты читай инструментом Read целиком; не подгружай файлы, не названные в вопросе.
- Протокол пиши инструментом Write в `decisions/meetings/`, покажи путь человеку.
- Роли НЕ запускаются как субагенты: субагенты изолированы и не имеют доступа к контексту друг друга —
  весь спор идёт в текущем контексте.
