# Data Objects — v1 (черновик)

Основание: PRD v2.3, требования FR-0..FR-5, FR-8, FR-9, FR-10.

---

## 1. SystemUser

Учётная запись преподавателя (единственного пользователя v1).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| username | string | Логин |
| password_hash | string | Хеш пароля |
| failed_attempts | int | Счётчик неудачных попыток входа |
| locked_until | datetime | Блокировка до (null = не заблокирован) |

**Обоснование:** FR-0 требует аутентификацию и блокировку после 5 неудачных попыток.

**Без неё невыполнимы:** FR-0.

---

## 2. Student

Студент курса, его репозиторий.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| full_name | string | ФИО |
| repo_url | string | URL публичного репозитория |
| git_host | enum(GitLab, GitHub) | Хостинг репозитория |
| added_at | datetime | Дата добавления в инструмент |
| is_active | bool | Активен (не удалён, репозиторий доступен) |

**Обоснование:** FR-1 — реестр студентов загружается из CSV; каждый привязан к адресу репозитория. `is_active` нужен для корректной обработки недоступных репозиториев (обход не срывается, студент помечается).

**Без неё невыполнимы:** FR-1, FR-4, FR-5, FR-9.

---

## 3. Lesson

Занятие курса — ось матрицы `репозиторий × занятие`.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| number | int | Порядковый номер занятия |
| title | string | Название занятия |

**Обоснование:** FR-2 задаёт чек-лист артефактов, привязанных к занятиям; FR-4 строит матрицу по этой оси. Отдельная сущность, а не атрибут `ArtifactDef`, позволяет менять порядок и названия занятий без правки каждого артефакта.

**Без неё невыполнимы:** FR-2, FR-4.

---

## 4. ArtifactDef

Определение ожидаемого артефакта: какой документ на каком занятии должен появиться.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| lesson_id | UUID | → Lesson.id |
| role | enum(interview, persona, user-story, prd, data-model, architecture, plan, code, tests) | Роль артефакта |
| expected_pattern | string | Глоб/путь для поиска файла (напр. `**/prd.md`) |
| template_relative_path | string | Путь в репозитории-шаблоне для сравнения (FR-4 «частично») |

**Обоснование:** FR-2 — конфиг артефактов не зашит в код; role используется и как метка в матрице, и как идентификатор для рёбер связности (EdgeDef). `template_relative_path` нужен для детекта заготовки = «частично» (BR-3).

**Без неё невыполнимы:** FR-2, FR-4, FR-5 (неизвестно, какие документы искать).

---

## 5. EdgeDef

Ребро связности: пара «роль источника → роль приёмника», подлежащая проверке.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| source_role | enum | Роль документа-источника |
| target_role | enum | Роль документа-приёмника |
| rubric_id | UUID | → Rubric.id |

**Обоснование:** FR-2 задаёт рёбра явно; FR-5 запускает LLM-проверку по каждому ребру. Рубрика вынесена в отдельную сущность, так как версионируется и переиспользуется.

**Без неё невыполним:** FR-5 (ядро не знает, какие пары сверять).

---

## 6. Rubric

Версионированное текстовое правило (инструкция) для LLM-агента.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| type | enum(edge, step) | `edge` — проверка ребра (v1); `step` — качество шага (v2) |
| version | string | Семантическая версия (напр. "1.0") |
| text | text | Полный текст инструкции агенту |
| created_at | datetime | Дата создания |

**Обоснование:** FR-2 хранит рубрики в конфиге; FR-5 требует версионирования для привязки вердикта к `(версия рубрики, модель)`. `type=step` зарезервирован для v2 (FR-11), не используется в v1.

**Без неё невыполним:** FR-5 (LLM-агент без инструкции).

---

## 7. GitCredential

Токен доступа к Git API хостинга.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| git_host | enum(GitLab, GitHub) | Хостинг |
| encrypted_token | string | Токен, зашифрованный при хранении (NFR-3, R-PD2) |

**Обоснование:** FR-3 требует токен для чтения репозиториев; NFR-3 и R-PD2 — шифрование. Одна запись на хостинг (курс использует два хостинга, §0).

**Без неё невыполнимы:** FR-3, FR-4, FR-5 (нет доступа к репозиториям).

---

## 8. SyncRun

Сессия автоматического обхода репозиториев.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| started_at | datetime | Начало обхода |
| completed_at | datetime | Окончание (null, если ещё идёт или упал) |
| status | enum(pending, in_progress, completed, partial, failed) | Итоговый статус |

**Обоснование:** FR-8 — обход по расписанию с инкрементальным чтением; FR-9 требует историю наблюдений с привязкой к коммитам. Каждый `SyncRun` — срез состояния на момент обхода. `partial` = часть репозиториев недоступна (NFR-2).

**Без неё невыполнимы:** FR-8, FR-9 (нет оси времени), FR-4 (нельзя показать «актуально на ЧЧ:ММ»).

---

## 9. ArtifactSnapshot

Результат проверки наличия артефакта у конкретного студента в конкретном обходе.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| sync_run_id | UUID | → SyncRun.id |
| student_id | UUID | → Student.id |
| artifact_def_id | UUID | → ArtifactDef.id |
| status | enum(found, partial, not_found) | FR-4: найден / частично (заготовка или неточное имя) / не найден |
| file_path | string | Путь к файлу в репозитории (null, если не найден) |
| source_commit_sha | string | SHA коммита, в котором найден файл (FR-9 устойчивость к force-push) |
| content_hash | string | Хеш содержимого (для инкрементального обхода, FR-8) |
| is_template_copy | bool | True, если содержимое совпадает с шаблоном (FR-4 «частично») |

**Обоснование:** FR-4 — матрица потребляет эти статусы; FR-9 — хронология на защите строится из последовательных снапшотов. `source_commit_sha` критичен: если студент переписал историю, данные не теряются.

**Без неё невыполнимы:** FR-4, FR-9.

## 10. CoherenceVerdict

Результат проверки связности по ребру для пары документов студента.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| sync_run_id | UUID | → SyncRun.id |
| student_id | UUID | → Student.id |
| edge_def_id | UUID | → EdgeDef.id |
| source_snapshot_id | UUID | → ArtifactSnapshot.id (версия документа A) |
| target_snapshot_id | UUID | → ArtifactSnapshot.id (версия документа B) |
| rubric_version | string | Версия рубрики на момент проверки |
| llm_model | string | Идентификатор модели LLM (напр. "gpt-4o") |
| verdict | enum(ok, break, pending) | Связность: нормально / разрыв / отложено из-за недоступности LLM |
| confidence | enum(high, medium, low) | Уверенность агента |
| entities_checked | int | Сколько сущностей источника проверено |
| entities_found | int | Из них отражено в приёмнике |
| entities_excluded | int | Из них явно исключено |
| entities_lost | int | Из них потеряно (разрыв) |
| points | json | ≤5 точек разрыва: каждая {entity, source_quote, what_searched_in_target} |
| notes | text | Заметка преподавателю (FR-5) |

**Обоснование:** FR-5 — ядро продукта; привязка к `(rubric_version, llm_model)` гарантирует стабильность вердикта (не мигает). `verdict=pending` — LLM недоступна, матрица работает (FR-5). FR-9 использует цепочку вердиктов по времени. FR-10 крепит override к конкретному вердикту.

**Без неё невыполнимы:** FR-5, FR-9 (на защите нечего показать), FR-10 (не к чему привязать отметку).

## 11. Override

Отметка ложного разрыва, проставленная преподавателем.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| coherence_verdict_id | UUID | → CoherenceVerdict.id |
| created_at | datetime | Когда поставлена отметка |
| reason | string | Причина (напр. «синоним», «переименование») |

**Обоснование:** FR-10 — преподаватель курирует находки. `reason` сохраняет контекст решения. Отметка обратима (удалить запись = снять отметку). При изменении пары документов (!= те же source_snapshot_id, target_snapshot_id) вердикт пересчитывается, старый override теряет привязку — это корректное поведение.

**Без неё невыполним:** FR-10.
