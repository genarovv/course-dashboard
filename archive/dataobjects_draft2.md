# Data Objects — draft 2

**Основание:** PRD v2.3 + решения CEO от 2026-07-02 (сессия проектирования схемы данных).
**Покрытие:** v1 (FR-0..FR-5, FR-8, FR-9, FR-10) + сущности под FR-11 (v2, схема закладывается заранее).
Связи между сущностями не проектируются — только сущности и атрибуты.

## Изменения относительно draft 1

1. **Сущность `Student` удалена, вместо неё `Repository`** (решение CEO): инструмент не хранит ФИО; сопоставление «репозиторий ↔ студент» преподаватель ведёт вне инструмента. Следствия:
   - FR-9 «карточка студента» → «карточка репозитория» (требует правки PRD).
   - **152-ФЗ снят с данных инструмента** (решение CEO 2026-07-02: логин в URL публичного репозитория не считаем ПДн; ФИО в инструменте нет) → R-PD3/R-PD4 выполняются тривиально, механизм обезличивания в схеме не нужен. Требует правки PRD §7 (сейчас там противоположная формулировка «адрес репозитория — потенциально ПДн»).
   - Метрика §4 «найти место разрыва на защите — секунды» получает ручной шаг (поиск репозитория по листочку).
2. **Добавлены сущности FR-11:** расширена `Rubric` (структурированные пункты, привязка рубрики шага к роли артефакта), добавлена `StepQualityCard`, обобщена `Override` (гасит и ложный разрыв, и ложное замечание).
3. Из `Repository` вынесен явный статус доступности (в draft 1 `is_active` смешивал «удалён» и «недоступен»); добавлена `default_branch` (негативный сценарий PRD §13: ветка ≠ `main`).
4. **Синхронизация с логической моделью** (`../data-model.md`, 2026-07-02): у `CoherenceVerdict` и `StepQualityCard` убраны `sync_run_id` (NFR-1: проверки асинхронны, завершаются вне обхода — вместо связи атрибут `computed_at`) и `repository_id` (транзитивно через снапшоты); `rubric_version`-строка заменена FK `rubric_id` (строка Rubric = конкретная версия, строки неизменяемы). Обоснования — data-model.md §5, В1–В3.

> **Единственный источник правды по сущностям данных — этот файл.** Draft 1 существовал в двух разошедшихся копиях (антипаттерн «две копии правды», ср. PRD §8); обе поглощены этим документом и перенесены в `../archive/` (`dataobjects_draft1-копия-корень.md`, `dataobjects_draft1-копия-product.md`) — не править и не цитировать как актуальные. Отсюда взято лучшее из обеих: трассировка к требованиям (product-копия) + явная nullability, `deferred` и подсказки реализации (корневая копия).

## 1. SystemUser

Учётная запись преподавателя (единственного пользователя v1).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| username | string | Логин |
| password_hash | string | Хеш пароля (bcrypt/argon2) |
| failed_attempts | int | Счётчик неудачных попыток входа |

**Обоснование:** FR-0 требует аутентификацию и блокировку на 15 минут после 5 неудачных попыток — счётчик и таймер надо хранить.

**Без неё невыполнимы:** FR-0, R-PD1.

## 2. Repository

Публичный git-репозиторий студента. Заменяет `Student` из draft 1: именных данных в инструменте нет.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| repo_url | string | URL публичного репозитория (не ПДн — решение CEO 2026-07-02) |
| git_host | enum(GitLab, GitHub) | Хостинг (курс идёт на двух, BR-5) |
| default_branch | string | Ветка по умолчанию (негативный сценарий §13: не всегда `main`) |
| added_at | datetime | Дата добавления (опоздавшие добавляются позже без потери истории, FR-1) |
| availability | enum(available, unavailable) | Доступность при последнем обходе (FR-1 сводка, FR-6 слепая зона, NFR-2) |

**Обоснование:** FR-1 — реестр загружается из CSV (берётся только колонка адресов; сводка «N доступно / M недоступно / K дубликатов» — дубликаты отсеиваются при импорте); FR-3/NFR-2 — недоступный репозиторий помечается, не срывая обход.

**Без неё невыполнимы:** FR-1, FR-3, FR-4, FR-5, FR-9 (карточка репозитория), FR-6.

| Атрибут | Тип | Описание |

**Обоснование:** FR-2 — конфиг не зашит в код; `role` — метка матрицы и идентификатор для рёбер (EdgeDef) и рубрик шага (FR-11).

**Без неё невыполнимы:** FR-2, FR-4, FR-5, FR-11 (неизвестно, какие документы искать и по какой рубрике шага гонять).

## 6. Rubric

Версионированное правило для LLM-агента. Расширена под FR-11: два вида рубрик.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| type | enum(edge, step) | `edge` — рубрика ребра (FR-5, v1); `step` — рубрика шага (FR-11, v2) |
| artifact_role | enum, null | Только для `type=step`: роль артефакта, которую проверяет рубрика (у `edge` — null, ребро задаёт пару само) |
| version | string | Семантическая версия (напр. "1.0") — входит в привязку вердикта |
| text | text | Полный текст инструкции агенту |
| items | json, null | Только для `type=step`: структурированные пункты чек-листа `[{item_id, text}]` — иначе «покрытию пунктов» в StepQualityCard не к чему привязаться |
| created_at | datetime | Дата создания версии |

**Обоснование:** FR-2 — рубрики версионируются; FR-5 и FR-11 привязывают вердикт к версии рубрики (защита от «мигания», регрессии — golden set).

**Без неё невыполнимы:** FR-5, FR-11.

## 9. ArtifactSnapshot

Наблюдение артефакта в конкретном репозитории в конкретном обходе. История, не состояние (§13).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| sync_run_id | UUID | → SyncRun.id |
| repository_id | UUID | → Repository.id |
| artifact_def_id | UUID | → ArtifactDef.id |
| status | enum(found, partial, not_found) | FR-4 |
| partial_reason | enum(inexact_name, wrong_place, template_copy), null | Причина «частично» (BR-3): неточное имя / не в том месте / заготовка |
| file_path | string, null | Фактический путь в репозитории |

**Без неё невыполнимы:** FR-4, FR-7, FR-8, FR-9, §13.

## 10. CoherenceVerdict

Результат проверки связности по ребру для пары документов репозитория (ядро, FR-5).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| edge_def_id | UUID | → EdgeDef.id |
| source_snapshot_id | UUID | → ArtifactSnapshot.id (версия документа A) |
| target_snapshot_id | UUID | → ArtifactSnapshot.id (версия документа B) |
| rubric_id | UUID | → Rubric.id (использованная версия рубрики; строка Rubric = версия) |
| llm_model | string | Идентификатор модели LLM |
| computed_at | datetime | Когда проверка завершилась (NFR-1: асинхронно, вне обхода — потому не FK на SyncRun) |
| verdict | enum(ok, break, deferred) | Цела / разрыв / отложено (LLM недоступна — матрица работает, FR-5) |
| confidence | enum(high, medium, low) | Уверенность; на защите разрыв подсвечивается только при `high` (риск §11) |
| entities_checked | int | Сущностей источника проверено |
| entities_found | int | Отражено (в т.ч. синонимом) |
| entities_excluded | int | Исключено явно |
| entities_lost | int | Потеряно (разрыв = ≥1) |
| points | json | ≤5 точек: `[{entity, source_quote, what_searched_in_target}]` |
| notes | text, null | Заметка преподавателю |

**Обоснование:** FR-5 — привязка вердикта к четвёрке `(content_hash A, content_hash B, rubric_id, llm_model)` исключает «мигание»; пересчёт только при изменении четвёрки. Репозиторий выводится транзитивно через снапшоты (правило целостности: оба снапшота одного репозитория).

**Без неё невыполнимы:** FR-5, FR-9, FR-10.

## 11. StepQualityCard (v2, FR-11)

Карточка соответствия артефакта рубрике шага. **Не балл и не оценка** (BR-2) — поля «score» нет и не появится.

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| snapshot_id | UUID | → ArtifactSnapshot.id (версия проверяемого документа) |
| rubric_id | UUID | → Rubric.id (type=step; использованная версия — строка Rubric = версия) |
| llm_model | string | Идентификатор модели LLM |
| computed_at | datetime | Когда прогон завершился (NFR-1, аналогично CoherenceVerdict) |
| status | enum(done, deferred) | `deferred` = LLM недоступна, аналогично FR-5 |
| coverage | json | Покрытие пунктов: `[{item_id, verdict: done/partial/missing}]` — item_id из Rubric.items |
| remarks | json | ≤5 замечаний: `[{quote, comment}]` — цитаты из документа |
| notes | text, null | Заметка преподавателю |

**Обоснование:** FR-11 наследует паттерн ядра: привязка к тройке `(версия документа, версия рубрики, модель)`, пересчёт только при изменении, регрессии — golden set. Межстуденческого сравнения нет (BR-4) — карточка живёт только внутри репозитория.

**Без неё невыполним:** FR-11. *Реализация за-гейчена входом в v2 (US + рубрики шагов + пилот точности, PRD §5.2) — в схеме заложена заранее, в код не идёт до выполнения условий.*

## 12. Override

Отметка ложной находки преподавателем. Обобщена под два вида находок (FR-10 + аналог для FR-11).

| Атрибут | Тип | Описание |
|---------|-----|----------|
| id | UUID | Первичный ключ |
| finding_type | enum(coherence, step_quality) | Что гасим: ложный разрыв (FR-10) или ложное замечание (FR-11) |
| finding_id | UUID | → CoherenceVerdict.id или StepQualityCard.id (по finding_type) |
| reason | string | Причина («синоним», «переименование», …) |
| created_at | datetime | Когда поставлена |
| revoked_at | datetime, null | Когда снята (FR-10: отметка обратима; снятие — не удаление записи, история решений сохраняется) |

**Обоснование:** FR-10 — единственное исключение из BR-1 (курирование, не ввод данных). Привязка к конкретному вердикту (который сам привязан к снапшотам) реализует правило «не подсвечивается, пока пара не изменилась»: новая версия пары → новый вердикт → отметка на него не распространяется. `revoked_at` вместо удаления — обратимость с аудируемой историей.

**Без неё невыполнимы:** FR-10, FR-11 (гашение ложного замечания).

## Чего в схеме сознательно нет

- **Student / ФИО** — решение CEO 2026-07-02: сопоставление «репозиторий ↔ студент» вне инструмента; ПДн в данных инструмента нет (логин в URL не считаем ПДн — то же решение), механизм обезличивания не нужен.
- **Матрица, слепая зона, «хроники»** — представления, производные от ArtifactSnapshot + Repository.availability, не таблицы.
- **Оценки/баллы** — запрещены BR-2 во всех версиях; отсутствие поля = исполнение правила на уровне схемы.
- **Golden set** — артефакт репозитория курса (`evals/`), не данные продукта.
- **Сущности студенческого интерфейса** — BR-4: их не существует by design.
