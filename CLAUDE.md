# CLAUDE.md — конституция проекта Course Dashboard

## Что это
Дашборд преподавателя курса «Управление агентной разработкой»: читает `.md`-артефакты из публичных репозиториев 25 студентов (GitLab/GitHub API), показывает матрицу «репозиторий × занятие» и подсвечивает **структурные разрывы связности** между артефактами. v1 — приватный однопользовательский инструмент (только преподаватель).
Это **демо-проект курса**: он же учебный пример, он же реальный инструмент. Разрабатывается агентно, live на занятиях.
**Стек:** не зафиксирован — сознательно вне PRD; решается на этапе архитектуры (занятие 7) → `ARCHITECTURE.md` + ADR-002.

## Железные правила
- **НИКОГДА** не закладывать ядро FR-5 (детект разрывов) в код, пока не пройдена Фаза 0 (живой custdev) — PRD §13. Операционализация ядра — ADR-001 (LLM-агент по правилам).
- **ВСЕГДА** прогонять golden set (`evals/golden-set.md`) при любой смене рубрики или модели ядра — расхождение с эталонным вердиктом = регрессия.
- **НИКОГДА** не добавлять студенческий или публичный интерфейс — BR-4: v1 teacher-only by design.
- **ВСЕГДА** spec-driven: сначала acceptance criteria конкретного FR/US, потом код.
- **ВСЕГДА** помечать выводы из синтетических интервью как гипотезы, не факты.
- Единственный канонический PRD — `product/prd.md`. Старые версии — только в `archive/`, их не править и не цитировать как актуальные.
- User stories канонически живут в `product/user-stories/` — PRD их не дублирует (защита от рассинхрона АП-14).
